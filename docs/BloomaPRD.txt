# 제품 요구사항 문서 (PRD)

## 1. 개요
AI 기반 웹 서비스 'Blooma’는 크리에이터가 복잡한 콘텐츠 기획 과정을 간단히 시각화·자동화할 수 있도록 지원한다. 핵심 가치는 “아이디어 → 스토리보드”를 몇 분 만에 끝내는 생산성 향상이다.

## 2. 문제 정의
- 기획 경험이 부족한 크리에이터는 아이디어를 체계화하기 어려움  
- 스토리보드 도구 다수가 영상·게임 중심으로 설계돼 범용성이 부족  
- 수동 작성 시 시간·노력이 과다 투입, 시각적 완성도도 낮음  
- 시장에 콘텐츠 기획 전용 AI 솔루션은 부재

## 3. 목표 및 목적
- 주요 목표: “AI 자동 스토리보드” 기능으로 기획 시간을 70% 단축  
- 보조 목표  
  - 기획 결과 시각 품질 향상  
  - 팀·클라이언트와 실시간 공유·피드백 간소화  
- 성공 지표 (임의)
  - MAU 10,000, 주간 재방문률 40% 이상  
  - AI 생성 스토리보드 완료율 80%  
  - 평균 페이지 로드 ≤ 1.5초

## 4. 대상 사용자
### 1차 사용자
- 20~40대 1인·소규모 크리에이터(유튜버, 틱톡커, 블로거)  
- 콘텐츠 기획 경험이 낮거나 시간 제약이 큰 사용자  
### 2차 이해관계자
- 광고주·브랜드(콘텐츠 발주 측)  
- MCN·에이전시 기획팀

## 5. 사용자 스토리
- “크리에이터로서, 아이디어 키워드만 입력해 AI가 씬별 스토리보드를 자동 완성해주길 원한다. 그래야 기획 시간을 줄일 수 있다.”  
- "크리에이터로서, 완성된 스토리보드 기획안을 시각화하여 광고주에게 전달한다. 그래야 광고주와 불필요한 커뮤니케이션을 줄일 수 있다.
- “협업 파트너로서, 웹 링크로 스토리보드를 열람·코멘트 달고 싶다. 그래야 피드백이 효율적이다.”  
- “브랜드 담당자로서, 제작자에게 수정 요청 내역을 한눈에 보고 싶다. 그래야 커뮤니케이션 비용을 줄인다.”

## 6. 기능 요구사항
### 핵심 기능
1. 카드 기반 시퀀스 생성 (Hook, Problem, Solution, Evidence, Benefit, CTA)
   - 입력: 각 카드별 제목, 본문 텍스트, 이미지(직접 업로드 또는 AI 생성)
   - 프로세스: 카드별 내용 작성 → 이미지 업로드/AI 생성 → 1080x1920 포맷 카드 완성
   - 완료 기준: 6개 카드 모두 이미지 및 텍스트 입력 완료 시
2. 카드 캔버스 시각화 UI
   - 드래그&드롭으로 카드 순서 변경
   - 카드 추가 및 삭제 기능 지원
   - 각 카드별 1080x1920 사이즈, 이미지(상단) + 본문(하단) 구조
   - 텍스트·이미지 개별 편집 지원
   - 카드별 버전 히스토리 및 미리보기(미정)
3. 내보내기  
   - PNG/PDF/MP4(슬라이드) 다운로드  
   - Google Drive 등 연동(미정)

### 부가 기능 (Nice-to-have)
- AI 아이디어 제안(사전에 Hook, Problem, Solution, Evidence, Benefit, CTA를 사용자에게 질문을 하고 답을 받은 후 AI로 기본 틀 마련)  
- 완성된 스토리보드를 생성된 이미지 기반으로 Image to Video 기능 추가 (확장 시)

## 7. 비기능 요구사항
- 성능: 생성 요청 30초 내 완료, UI 반응 < 200ms  
- 보안: Supabase Row Level Security, OAuth 2.0  (미정) [Clerk 대체 가능]
- 사용성: Figma UX 가이드라인 준수, WCAG 2.1 AA  
- 확장성: 멀티테넌트 구조, 월 100만 호출 대응  (미정)
- 호환성: 크로스브라우저(Chrome, Safari, Edge 최신 2버전)

## 8. 기술 고려사항 (미정)
- 프론트엔드: Next.js 15, TypeScript, Zustand
- 백엔드: Supabase(PostgreSQL, Edge Functions), Cloudinary(For images)
- AI: 텍스트->OpenAI GPT-4o,  Gemini / 이미지-> Replicate, Gemini imagen, Stable Diffusion, HiDream-l1, Flux.1, DeepFloyd IF, Free-Image-Generation-CC0(허깅페이스), InvokeAI, stability.ai, GPT DALL·E 등
- 아키텍처: Serverless + Vercel, Supabase Storage CDN 
- 통합: Google Drive
- 데이터: 프로젝트·사용자·버전 테이블, 180일 보관 후 백업

## 9. 성공 지표 (미정)
- 사용자 지표: 가입→첫 스토리보드 생성 전환률 60%  
- 비즈니스 지표: 월 구독 매출 5만 달러, Churn < 5%  
- 기술 지표: 에러율 < 0.5%, 99.9% 가용성

## 10. 일정 및 마일스톤 (미정)
- Phase 1 (M0~M2) MVP  
  - AI 기본 생성, 캔버스 UI, PDF 내보내기  
- Phase 2 (M3~M5) 협업 & 결제  
  - 댓글, 실시간 동기화, Toss payments, paypal 구독 
- Phase 3 (M6~M8) 확장  
  - 템플릿 마켓, 음성 입력, SEO 태깅, 모바일 PWA

## 11. 리스크 및 대응 (미정)
- AI 생성 품질 편차 → 지속적 프롬프트 튜닝, 사용자 평가 루프  
- 모델 호출 비용 증가 → 캐싱·Token 최적화  
- 저작권 이슈 → 이미지 생성 시 라이선스 필터 적용  
- 초기 트래픽 급증 → Vercel Pro, Supabase Auto Scaling 확보

## 12. 향후 고려 (미정)

- 자체 파인튜닝 모델 구축으로 비용 30% 절감  
- 크리에이터 분석 대시보드, 광고 브리프 자동 매칭  
- 오프라인 워크숍·API SaaS 확장
